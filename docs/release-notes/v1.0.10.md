# ConfigPilot v1.0.10

本版本新增 Claude Desktop 配置与官方安装能力，并优化导航图标、状态区排版和模型获取提示。

## 新功能

- 新增 Claude Desktop 独立页面，可分别切换 Developer Mode 与 Third-Party Inference Gateway。
- 支持配置兼容 Anthropic `/v1/messages` 的 Gateway endpoint、认证方式、模型和额外 Header。
- 安装下拉菜单内置 Claude Code CLI 与 Claude Desktop 官方安装流程，覆盖 Windows、macOS 和 Ubuntu/Debian。
- Windows Claude Desktop 使用 Anthropic 官方独立安装包，不使用 Microsoft Store。
- 安装过程支持异步下载、进度显示和下载阶段取消。
- 官方下载源启用主机白名单；Windows 安装包校验 Anthropic 发布者签名，Linux DEB 校验 SHA-256。

## 界面优化

- Codex 与 Claude 导航项使用对应产品图标。
- Claude 状态区改为紧凑安装按钮，其他状态严格等宽排布。
- 模型获取成功通知不再展开完整模型列表，避免提示条超宽。

## macOS 首次安装须知

当前 macOS 版本仅支持 Apple Silicon（M 系列芯片），不支持 Intel Mac。

维护者目前没有 Apple Developer Program 会员，因此 DMG 尚未使用 Developer ID 签名和 Apple 公证。macOS Gatekeeper 可能显示“ConfigPilot 已损坏，无法打开”，但这不代表 DMG 文件真的损坏。

请先把 `ConfigPilot` 拖入 `Applications`，再打开“终端”执行：

```bash
xattr -dr com.apple.quarantine "/Applications/ConfigPilot.app"
open "/Applications/ConfigPilot.app"
```

同一已安装版本通常只需执行一次。请仅从本项目官方 Releases 下载。
