# ConfigPilot v1.0.12

本版本加入应用内自动更新，并修复 OpenAI 兼容接口地址缺少 `/v1` 的问题。

## 主要更新

- 启动后自动检查 GitHub Releases，帮助页也可手动检查更新。
- Windows 支持在应用内下载新版、显示下载进度、静默安装并自动重启。
- macOS 检测到新版后打开官方 Release 页面，避免在未签名、未公证状态下错误执行静默覆盖。
- Codex `base_url` 和 Claude Gateway endpoint 缺少 `/v1` 时自动补全；已有 `/v1` 时不会重复追加。
- Codex 获取模型时统一请求规范化后的 `/v1/models`。
- 更新配置集中到 `app_config.json`，打包流程会携带运行时版本与更新参数。
- PrismQML 更新至 `0.2.24.13`，使用更严格的更新响应解析和原子下载事务。

## 升级说明

ConfigPilot v1.0.11 尚未包含应用内更新器，因此需要手动安装 v1.0.12 一次。安装 v1.0.12 后，后续版本可直接在应用内检查、下载并升级。

已有 Claude Gateway 配置不会在程序启动时被静默改写。打开 ConfigPilot 并重新点击一次“应用”，即可把缺少版本路径的现有 endpoint 规范化为 `/v1`。

## macOS 首次安装须知

当前 macOS 版本仅支持 Apple Silicon（M 系列芯片），不支持 Intel Mac。

维护者目前没有 Apple Developer Program 会员，因此 DMG 尚未使用 Developer ID 签名和 Apple 公证。macOS Gatekeeper 可能显示“ConfigPilot 已损坏，无法打开”，但这不代表 DMG 文件真的损坏。

请先把 `ConfigPilot` 拖入 `Applications`，再打开“终端”执行：

```bash
xattr -dr com.apple.quarantine "/Applications/ConfigPilot.app"
open "/Applications/ConfigPilot.app"
```

同一已安装版本通常只需执行一次。请仅从本项目官方 Releases 下载。
