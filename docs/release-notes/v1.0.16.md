# ConfigPilot v1.0.16

本版本将内置界面引擎升级至 PrismQML 0.3.1.4，补齐仍可能阻塞界面的系统调用，并重新整理 Claude Desktop 配置页的字段网格和敏感配置操作。

## 主要更新

- PrismQML 从 `0.3.1.1` 升级至 `0.3.1.4`，改进锚定弹层定位、导航标题显示和图表按需加载，并加入统一的异步 QML 页面加载能力。
- 更新安装程序启动、系统目录打开、外部发布页打开和模型档案初始化进一步移出 GUI 主线程。
- Claude Desktop 网关与高级配置改为等宽双列字段网格，标签、说明和输入控件使用统一视觉节奏。
- 敏感密钥与请求头的清除操作改为“应用时删除”复选框，仅在存在已保存内容时显示，避免把草稿操作误解为即时开关。
- 强化后台下载、安装启动和对象销毁期间的生命周期保护，应用关闭路径不再等待后台线程。

## 升级说明

ConfigPilot 已包含应用内更新器。Windows 用户可在应用内完成下载、静默安装和重启；macOS 用户检测到新版后会打开官方 Release 页面。

本版本更新了内置 PrismQML 引擎，必须安装 `v1.0.16` 的新程序包；仅更新配置文件不会获得新的界面引擎。

## 下载

- macOS Apple Silicon：`ConfigPilot_1.0.16_arm64.dmg`

## macOS 首次安装须知

当前 macOS 版本仅支持 Apple Silicon（M 系列芯片），不支持 Intel Mac。

维护者目前没有 Apple Developer Program 会员，因此 DMG 尚未使用 Developer ID 签名和 Apple 公证。macOS Gatekeeper 可能显示“ConfigPilot 已损坏，无法打开”，但这不代表 DMG 文件真的损坏。

请先把 `ConfigPilot` 拖入 `Applications`，再打开“终端”执行：

```bash
xattr -dr com.apple.quarantine "/Applications/ConfigPilot.app"
open "/Applications/ConfigPilot.app"
```

同一已安装版本通常只需执行一次。请仅从本项目官方 Releases 下载。
