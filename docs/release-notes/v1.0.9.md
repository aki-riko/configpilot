# ConfigPilot v1.0.9

本版本修复 Windows 旧版本升级后的安装目录和任务栏图标问题。

## 主要修复

- 旧版 `CodexConfig` 用户升级时，安装目录迁移为 `ConfigPilot`。
- 开始菜单程序组从“Codex 配置助手”迁移为 `ConfigPilot`。
- Windows 快捷方式写入与 PrismQML 一致的 `PrismQML.ConfigPilot` AppUserModelID。
- 在创建 QML 窗口前设置应用级 ICO，避免任务栏缓存通用窗口图标。
- 保留原 AppId，继续支持从旧版本直接升级。

## 下载

- Windows：`ConfigPilot_Setup_1.0.9.exe`
- macOS Apple Silicon：`ConfigPilot_1.0.9_arm64.dmg`

## macOS 首次安装须知

当前 macOS 版本仅支持 Apple Silicon（M 系列芯片），不支持 Intel Mac。

维护者目前没有 Apple Developer Program 会员，因此 DMG 尚未使用 Developer ID 签名和 Apple 公证。macOS Gatekeeper 可能显示“ConfigPilot 已损坏，无法打开”，但这不代表 DMG 文件真的损坏。

请先把 `ConfigPilot` 拖入 `Applications`，再打开“终端”执行：

```bash
xattr -dr com.apple.quarantine "/Applications/ConfigPilot.app"
open "/Applications/ConfigPilot.app"
```

同一已安装版本通常只需执行一次。请仅从本项目官方 Releases 下载。
